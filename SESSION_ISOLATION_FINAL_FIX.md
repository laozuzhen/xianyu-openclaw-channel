## 🔧 修复内容

### 1. 修复缓存键冲突（问题1 + 问题3）

**问题描述**:
- 原缓存键：`route.sessionKey`
- 在闲鱼场景下，多个买家咨询同一商品时 sessionKey 相同
- 导致买家 A 的消息被买家 B 的消息覆盖

**修复方案**:
```typescript
// 修改前（第 168 行）
const cacheKey = route.sessionKey;

// 修改后
const cacheKey = `${route.sessionKey}:${senderId}:${data.messageId}`;
```

**修复理由**:
- `route.sessionKey` - 确保不同对话隔离
- `senderId` - 确保同一对话中不同用户隔离
- `data.messageId` - 确保同一用户的不同消息隔离（解决并发竞态条件）

**影响范围**: 
- ✅ 解决多买家串台问题
- ✅ 解决并发竞态条件（问题5）

```python
async def stream_messages(
    request: Request,
    account_id: str = "default",
    last_event_id: Optional[str] = Header(None, alias="Last-Event-ID"),
):
    """SSE 端点：持续推送指定账号的入站消息"""

        try:
```

## ❌ 问题诊断

### 当前问题
**SSE 消息发送时没有检查商品所有权**

1. **消息发布阶段** (`XianyuAutoAsync.py` 第 7793 行)
   - 所有收到的买家消息都会被发布到 `bridge_queue`
   - **没有检查** `item_id` 是否属于当前账号
   - 导致其他账号的商品消息也会被推送

2. **SSE 推送阶段** (`bridge_api.py` 第 105 行)
   - 直接从队列读取消息并推送
   - **没有任何过滤逻辑**

### 影响范围
- OpenClaw Channel Plugin 会收到不属于自己账号的商品消息
- 可能导致误操作或混淆
- 浪费网络带宽和客户端资源

---

## ✅ 解决方案

### 方案 A：在消息发布前过滤（推荐）

**优点**:
- 在源头过滤，避免无效消息进入队列
- 减少队列内存占用
- 逻辑集中，易于维护

**实现位置**: `XianyuAutoAsync.py` 第 7773-7795 行

**修改代码**:

```python
# 🔗 OpenClaw Bridge 模式：将买家消息发布到 Bridge 消息队列
if BRIDGE_ENABLED and bridge_queue:

        # ✅ 新增：检查商品是否属于当前账号
```

### 方案 B：在 SSE 推送前过滤

**优点**:
- 不修改消息发布逻辑
- 可以在推送端统一处理

**缺点**:
- 无效消息仍会进入队列
- 需要在推送时查询数据库，增加延迟

**实现位置**: `bridge_api.py` 第 75-120 行

**修改代码**:

```python
@bridge_router.get("/messages")
async def stream_messages(
    request: Request,
    account_id: str = "default",
    last_event_id: Optional[str] = Header(None, alias="Last-Event-ID"),
):
    """SSE 端点：持续推送指定账号的入站消息"""
    queue = bridge_queue.subscribe(account_id)
    
    async def event_generator():
        try:
            # 如果有 Last-Event-ID，先补发断线期间的消息
            if last_event_id:
                missed = bridge_queue.get_missed_messages(account_id, last_event_id)
                for msg in missed:
                    # ✅ 新增：过滤不属于当前账号的商品消息
                    if await _should_send_message(msg, account_id):
                        eid = msg.get("event_id", "")
                        data = json.dumps(msg, ensure_ascii=False)
                        yield f"id: {eid}\nevent: message\ndata: {data}\n\n"
            
            # 持续监听队列
            while True:
                if await request.is_disconnected():
                    break
                
                try:
                    msg = await asyncio.wait_for(queue.get(), timeout=30.0)
                    
                    # ✅ 新增：过滤不属于当前账号的商品消息
                    if await _should_send_message(msg, account_id):
                        eid = msg.get("event_id", "")
                        data = json.dumps(msg, ensure_ascii=False)
                        yield f"id: {eid}\nevent: message\ndata: {data}\n\n"
                except asyncio.TimeoutError:
                    yield ": keepalive\n\n"
        finally:
            bridge_queue.unsubscribe(account_id, queue)
    
    return StreamingResponse(
        event_generator(),
        media_type="text/event-stream",
        headers={
            "Cache-Control": "no-cache",
            "Connection": "keep-alive",
            "X-Accel-Buffering": "no",
        },
    )


async def _should_send_message(msg: dict, account_id: str) -> bool:
    """检查消息是否应该发送（商品是否属于当前账号）"""
    try:
        # 如果消息中没有 itemId，默认发送（可能是系统消息）
        item_id = msg.get("itemId")
        if not item_id or item_id == "未知商品":
            return True
        
        # 检查商品是否属于当前账号
        from db_manager import db_manager
        item_info = db_manager.get_item_info(account_id, item_id)
        if not item_info:
            logger.debug(f"[Bridge] 过滤消息: 商品 {item_id} 不属于账号 {account_id}")
            return False
        
        return True
    except Exception as e:
        logger.warning(f"[Bridge] 检查消息归属失败: {e}，默认发送")
        return True  # 出错时默认发送，避免丢失消息
```

## 🎯 推荐实现方案

**推荐使用方案 A（在消息发布前过滤）**

### 理由
1. **性能更好**: 在源头过滤，避免无效消息进入队列
2. **逻辑一致**: 与现有的自动发货逻辑保持一致
3. **易于维护**: 所有商品归属检查逻辑集中在 `XianyuAutoAsync.py`
4. **减少资源消耗**: 队列中只保存有效消息
